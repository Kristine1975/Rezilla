Sélectionneurs de ressources

Depuis la version 1.1, Rezilla a introduit des sélectionneurs de ressources (pickers): un sélectionneur est une fenêtre spéciale qui affiche graphiquement toutes les ressources appartenant à un type particulier. L'idée des sélectionneurs est de permettre de prévisualiser le contenu des ressources.

La manière dont les ressources sont affichées dans les fenêtres de sélectionneur dépend de leur type. Par exemple, un sélectionneur pour les ressources de type 'icns' affiche l'onglet (icone de dimensions 128x128 pixels) contenu dans ce type de ressources. Un sélectionneur pour les ressources 'TEXT' affichera le début du texte, à la manière de ce que font les dialogues de navigation lorsque l'on navigue dans le système pour ouvrir un fichier. Le numéro de la ressource est par ailleurs toujours indiqué.

Il n'existe pas forcément de sélectionneur pour un type de ressources donné. Les sélectionneurs n'ont d'intérêt que pour les ressources dont le contenu peut être visualisé d'une manière ou d'une autre (texte, icones, images, motifs, curseurs, menus, etc.).

Pour ouvrir un sélectionneur de ressources pour un type particulier, il suffit de faire un double-clic sur le nom du type dans les documents contenant les tables de ressources.

Lorsqu'une fenêtre de sélectionneur est ouverte, on peut sélectionner une ressource au moyen d'un simple clic de souris. La ressource ainsi sélectionnée est entourée d'un cadre rouge: un simple clic suffit ensuite pour la déselectionner. Lorsqu'une ressource est sélectionnée, les commandes du menu Resources sont activées afin de permettre d'éditer, de dupliquer ou de supprimer la ressource en question. Un double-clic sur une ressource est équivalent à la commande Edit du menu Resources.

Actuellement, Rezilla définit des sélectionneurs de ressources pour les types suivants: 'TEXT', 'MENU', 'STR#', 'MBAR', 'RID#', 'Mcmd', 'RidL', 'cicn', 'ICON', 'SICN', 'icns', 'PICT', 'PAT ', 'ppat', 'PAT#', 'ICN#', 'icl4', 'icl8', 'icm#', 'icm4', 'icm8', 'ics#', 'ics4', 'ics8', 'CURS', 'crsr'
