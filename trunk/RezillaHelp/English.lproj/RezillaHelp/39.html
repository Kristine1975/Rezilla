Templates panel

In the Templates panel, one can select the symbol to use to represent hexadecimal values: either $ or 0x. This is the symbol which will be used by the template editor when it displays a hex number. If you write an hexadecimal value in a field, you can use any of these symbols indifferently: the editor will accept both.

Hexadecimal letters (a to f) can be displayed in lowercase or in uppercase form. This is set using the Hexadecimal digits preference.

If the Display filler fields in templates checkbox is on the Template editor will display all the tags corresponding to filling data (see the subsection Phantom fields in the Templates Syntax section below). Otherwise these values are hidden. Furthermore, if the Enable filler fields in templates checkbox is on, these fields will be editable, otherwise they will be disabled and displayed in grey.

Since version 1.0.8 of Rezilla, one can choose the format to display the four values of a type RECT tag: top left bottom right (coordinates of the top left and the bottom right corners) or top left width height (coordinates of the top left corner followed by width and height).
