Préférences

Rezilla permet de fixer diverses préférences. On accède aux préférences au moyen de la commande Préférences dans le menu de l'application (menu Rezilla) ou au moyen du raccourci clavier Command-, (taper la touche virgule en maintenant la touche commande enfoncée). Un dialogue multipage s'affiche comportant trois panneaux:
  • le panneau General
  • le panneau Export
  • le panneau Compare
  • le panneau Interface
  • le panneau Templates
  • le panneau Editors
  • le panneau Misc

Les préférences sont sauvegardées sur disque dans le fichier de préférences de l'utilisateur uniquement lorsque l'on quitte l'application. Elles sont stockées dans un fichier nommé net.sourceforge.rezilla.plist situé dans le dossier de préférences de l'utilisateur.

Panneau General

Dans le panneau General, on peut fixer le nombre maximal d'articles contenus dans le menu Recent Items (par défaut 10). Un bouton Reset permet d'effacer la liste actuelle. On peut aussi spécifier la branche qui sera utilisée par défaut pour créer une nouvelle table de ressources (cela correspond aux boutons radios affichés dans le dialogue New Map: ce choix par défaut peut être modifié plus tard dans le dialogue).

Panneau Export

Dans le panneau Export, on peut choisir la DTD à utiliser pour créer un export XML (voir à la section Export dans les fichiers d'aide). On peut également spécifier l'encodage à utiliser pour les données binaires: encodage Base64 ou représentation hexadécimale. La case à cocher Include binary data in output permet de spécifier si les données binaires de chaque ressource doivent être inclues dans les fichiers d'export: cela concerne pour le moment uniquement les formats Texte et XML, pas HTML.

Dans le champ appelé Text editor signature, on peut indiquer le créateur d'un éditeur de textes à attribuer aux fichiers produits afin que, par la suite, ceux-ci soient automatiquement ouverts dans cet éditeur lorsqu'on fait un double-clic sur leur icone.

Panneau Compare

Dans le panneau Compare, on peut fixer des valeurs par défaut pour les critères de comparaison apparaissant dans le dialogue Compare Maps: ces valeurs peuvent être modifiées par la suite dans le dialogue lui-même. On peut aussi fixer une valeur par défaut pour les boutons radios apparaissant au bas de la fenêtre Compare Results affichant les résultats d'une comparaison de tables de ressources et indiquant le format à utiliser pour afficher les données binaires des ressources (hexadécimal ou texte): ces valeurs peuvent êtremodifiées par la suite lorsque la fenêtre est affichée.

Panneau Interface

Dans le panneau Interface, on peut fixer la police et la taille utilisées pour afficher les représentations hexadécimales ou textuelles des données de ressources dans les fenêtres de l'éditeur hexadécimal ou dans les fenêtres de résultats de comparaison. Noter que seules les polices à espacement fixe (telles que Monaco ou Courier) garantissent un affichage en colonnes verticalement alignées.

Panneau Templates

Dans le panneau Templates, on peut choisir le symbole à utiliser pour représenter les nombres hexadécimaux: $ ou 0x. C'est le symbole qui est utilisé par l'éditeur de gabarits lorsqu'il doit afficher une valeur hexadécimale. Inversement si l'on doit saisir un nombre hexadécimal dans un champ, on peut utiliser indifféremment l'un des deux symboles. Rezilla accepte les deux.

Les lettres hexadécimales (a à f) peuvent être affichées en majuscules ou en minuscules. On fait ce choix au moyen de la préférence appelée Hexadecimal digits.

Si la case à cocher appelée Display filler fields in templates est sélectionnée, l'éditeur de gabarits affichera les données correspondant aux balises de remplissage (voir la sous-section Champs fantômes dans la section Syntaxe des gabarits). Autrement, ces valeurs seront cachées. De plus, si la case à cocher appelée Enable filler fields in templates est sélectionnée, ces champs seront rendus éditables, autrement ils sont désactivés et apparaissent en gris.

Panneau Editors

Dans le panneau Editors, on peut spécifier des propriétés concernant certains éditeurs particuliers.

Préférences de l'éditeur d'images

La préférence Use 8-bit colors est utilisée pour des conversions d'images couleur. Elle force à utiliser une profondeur de couleur de 8 au lieu de 32.

La préférence Use Full Color Tables indique d'utiliser des palettes de 256 couleurs. Autrement la palette Apple classique avec seulement 32 couleurs est utilisée.

Préférences de l'éditeur Unicode

Si la préférence Do Font Substitution est activée, Rezilla affichera automatiquement le texte Unicode en se servant des polices de caractères appropriées correspondant aux scripts utilisés dans la ressource.

La préférence Insert Byte Order Mark dit s'il faut inclure la marque d'ordre (BOM, de code-point U+FEFF) au début de la ressource. Ce symbole Unicode sert souvent à identifier des données Unicode au format UTF-16 qui est la forme d'encodage utilisée par Rezilla.

Panneau Misc

Dans le panneau Misc (abréviation de miscellaneous qui signifie divers), on peut spécifier au moyen de la préférence Set type and creator when closing un type et une signature à attribuer à un fichier de ressources après qu'il a été édité par Rezilla et refermé. Les valeurs par défaut sont rsrc et Rzil respectivement. Cette préférence est à utiliser avec précaution car certains fichiers, tels que les fichiers d'applications, possèdent leur propre signature qui ne doit pas être modifiée. Il y a en outre une case à cocher appelée Only files with .rsrc extension qui permet de limiter la portée de cette préférence aux fichiers portant une extension .rsrc.

La préférence Use Rezilla's signature for new maps permet de décider si Rezilla doit attribuer sa propre signature aux fichiers de ressources nouvellement créés. S'ils ont cette signature, ils seront, par la suite, ouverts automatiquement dans Rezilla en réponse à un double clic sur leur icone.
