Exporting/importing a resource map

The Export Map command of the File menu lets you generate a file containing the data of the resource map (all resources of all types, with their ID, name, attributes etc.) in various formats. Available formats are:

  • Text
  • HTML
  • XML
  • DeRez

In Text and XML formats, it is possible to decide to include the binary data for each resource or not: see the Export panel of the Preferences dialog. In DeRez format, the resource data are always included. On the contrary, in HTML format, the resource data are never included.

In Text format, the binary code for each resource is encoded either in Base64 encoding (which is the format used on MacOSX for many other types of binary data stored in XML files, like for instance preferences files, property lists etc.), or in hexadecimal representation. This is a choice which can be specified in the Export panel of the Preferences dialog. In XML format, the binary data are always Base64-encoded. In DeRez format, the binary data are always Hex-encoded.

DeRez is a format which is of interest only for developers: it is the format used by the Rez and DeRez tools. The resources are output as data in a way similar to what the DeRez tool produces when no specific resource pattern is specified.

For XML export, the representation of the data is based on a DTD (Document Type Definition) defined in the file Rezilla.dtd included in Rezilla's distribution. This is an elements-based DTD in which all the properties are represented by a specific tag. and an attributes-based DTD in which most of the properties are represented as attributes for the basic type and resource elements.

Here is an example of the XML output produced by the Export Map command: 

	<Type>
		<TypeCode>ALRT</TypeCode>
		<ResourcesArray>
			<Resource>
				<ResourceID>204</ResourceID>
				<ResourceName>Low Memory Warning</ResourceName>
				<ResourceFlags>20</ResourceFlags>
				<ResourceSize>14</ResourceSize>
				<ResourceData>
				AGgAggDNAegAzEREMAo=
				</ResourceData>
			</Resource>
		</ResourcesArray>
	</Type>
	<Type>
		<TypeCode>DITL</TypeCode>
		<ResourcesArray>
			<Resource>
				<ResourceID>204</ResourceID>
				<ResourceName></ResourceName>
				<ResourceFlags>20</ResourceFlags>
				<ResourceSize>120</ResourceSize>
				<ResourceData>
				AAEAAAAAAEcBIABbAVwEAk9LAAAAAAAKAEsAOgFciFdNZW1v
				cnkgaXMgZ2V0dGluZyBmdWxsLiBQbGVhc2UgdHJ5IHRvIGFs
				bGV2aWF0ZSB0aGUgcHJvYmxlbSBieSBjbG9zaW5nIHNvbWUg
				ZG9jdW1lbnRzLiDl
				</ResourceData>
			</Resource>
		</ResourcesArray>
	</Type>

The Import Map command of the File menu can be used to import a resource map previously exported in XML format with the Export Map command. The import file must be an XML file conforming to the Rezilla DTD.
