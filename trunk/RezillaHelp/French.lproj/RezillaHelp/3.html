Tables de ressources

À l'ouverture d'un fichier de ressources, la table de ressources est affichée dans une fenêtre donnant la liste des divers types de ressources contenus dans le fichier: si l'on clique sur le petit triangle placé à la gauche du type, la table affiche, au second niveau, la liste des différentes ressources de ce type, avec leur numéro (ID), leur taille et leur nom (si elles en ont un).

Le bas de la table de ressources comporte des indicateurs affichant le nombre de types différents contenus dans cette table et le nombre total de ressources. Les lettres DF ou RF à droite du nombre de ressources indiquent si la table de ressources a été ouverte à partir de la branche de données (DF signifie data fork) ou de la branche de ressources du fichier (RF signifie resource fork).

Le coin inférieur gauche de la fenêtre affiche une petite icone représentant un verrou: si le verrou est fermé, la table de ressources est en mode lecture seule. Dans ce mode, on ne peut exécuter aucune commande d'édition: toutes les commandes de menu relatives aux fonctions d'édition de la table de ressources sont désactivées. On peut cependant visualiser le contenu d'une ressource au moyen de l'éditeur hexadécimal mais il sera en mode lecture seule lui aussi.

Éditer une ressource

En double-cliquant sur une ressource dans la table, on appelle un éditeur pour cette ressource. Cela équivaut à selectionner la ressource et choisir la commande Edit dans le Menu Ressource. Il existe principalement trois modes d'édition pour un type de resource particulier faisant appel à trois types d'éditeurs:
  • un éditeur dédié
  • l'éditeur de gabarits
  • l'éditeur hexadécimal

Quelques types de ressources possèdent un éditeur dédié: c'est en général un éditeur qui offre une interface graphique spécifique permettant d'éditer et de modifier facilement la ressource sans avoir à connaître quoi que ce soit de la façon dont elle est construite et stockée. L'éditeur de gabarits est aussi un éditeur graphique. Il fait appel à ce que l'on appelle un gabarit de ressource (en anglais template): un gabarit est une description abstraite de la structure d'une ressource qui est utilisée par Rezilla pour construire une interface graphique. De très nombreuses ressources possèdent un format prédéfini qui peut être décrit au moyen d'un gabarit. Rezilla possède plus de 150 gabarits et il est aussi possible d'installer des gabarits supplémentaires qui seront automatiquement pris en compte par Rezilla (voir la section Éditeur de Gabarits plus loin). L'éditeur hexadécimal enfin est disponible pour tous les types de ressources: les données binaires de la ressource sont éditées directement par leur valeur hexadécimale. C'est en quelque sorte un éditeur de bas niveau qui permet d'accéder directement au contenu binaire de la ressource (voir la section Éditeur Hexadécimal plus loin).

Lorsque la commande Edit est invoquée pour une ressource particulière, Rezilla recherche tout d'abord s'il existe un éditeur dédié pour ce type de ressource. S'il n'en trouve aucun, il cherche alors si un gabarit est disponible afin d'invoquer l'éditeur de gabarits. En cas d'échec, il appellera en dernier recours l'éditeur hexadécimal.

Types de remplacement

Certains types de ressource ont des structures identiques. Par exemple, une ressource de type kcs4 décrivant une petite icone de clavier utilise le même format qu'une ressource de type ics4. Ces types sont donc interchangeables du point de vue des éditeurs, l'un pouvant être édité au moyen d'un éditeur de l'autre. Rezilla connaît un certain nombre de situations de ce type et opère la substitution automatiqument lorsque c'est nécessaire. Par exemple, s'il ne trouve pas de gabarit pour une ressource itl0, il utilisera celui des ressources INTL à la place.

On peut aussi forcer Rezilla à se servir d'un type à la place d'un autre. Il faut pour cela utiliser la commande Save as type... qui se trouve dans le menu Resource. Elle affiche un petit dialogue dans lequel on peut spécifier le type de remplacement soit à partir d'un menu, soit en le tapant directement dans le champ de saisie.

Modificateurs

Lorsque l'on fait un double-clic sur une ressource dans la table, on peut aussi enfoncer simultanément une touche modificatrice:

  • si la touche Majuscule est enfoncée pendant un double-clic, l'éditeur de gabarits est appelé directement pour cette ressource. Si aucun gabarit n'est disponible, l'éditeur hexadécimal est appelé à la place.
  • si la touche Option est enfoncée pendant un double-clic, l'éditeur hexadécimal est appelé directement pour cette ressource.

Plusieurs ressources peuvent être sélectionnées simultanément dans la table afin de les éditer.

En cliquant (simple clic) sur une ressource avec la touche Contrôle enfoncée, on appelle la fenêtre de l'Inspecteur pour cette ressource. Cela équivaut à selectionner la ressource et choisir le commande Get Info dans le Menu Ressource. Voir la section fenêtre de l'Inspecteur plus loin.

Noter que cliquer sur un des triangles avec la touche Option enfoncée permet d'agir sur tous les types de ressources à la fois: si le triangle était fermé, tous les types de ressources seront ouverts, et inversement si il était ouvert alors tous les types de ressources seront fermés.
