Template Editor

Rezilla is able to parse resources using descriptive templates and to build a graphical interface out of it, allowing to easily edit all the elements of the resource. For instance, the following image shows how Rezilla would edit a Txtr resource containing font and size information for a document.

There are three kinds of locations where TMPL resources can be found. They are described in details in the following subsections:

  • internal templates are predefined templates stored inside the application. Rezilla ships with more than 360 predefined templates covering the most currently used resource types;
  • external templates are additional templates that one can add to one of the Application Support folders;
  • local templates are TMPL resources present in the currently edited resource map.

When Rezilla attempts to edit a resource using a template, it looks for the appropriate template in all these locations in the following order: local, external, internal. You can thus override the definition of an internal template by putting a new definition in the external locations.

Templates editor windows have three buttons:

  • the Cancel button lets you abort the editing and close the window without asking if modifications should be saved.
  • the Validate button lets you save your modifications and close the window.
  • the Revert button lets you discard the recent modifications and revert to the last saved version of the resource in the resource map in memory (not the last saved version of the resource on disk).

Note that pressing the Enter key (not the Return key) is equivalent to clicking on the Validate button.

If the Close button (the leftmost red little button in the title bar) is clicked and if any of the resource fields has been edited, Rezilla will ask you if you want to save your modifications. If the answer is yes, the modified resource is installed in the resource map. This does not mean though that the new data are saved to the disk: this happens only when you close the resource map table itself or invoke the Save menu item when this resource map table is at the front.

Lists of elements in templates

Some resources contain lists of data, for instance a list of strings, a list of graphical objects etc. Templates are able to handle various kinds of lists: as a header of each list, the Templates editor displays a count of the items currently in the list and two buttons with a Minus and a Plus sign respectively. Typically you'll use the Minus button to remove an item from the list and the Plus button to add a new item.

A list item can contain different fields: they are all displayed in a common frame which it is very easy to identify. Clicking on one such item selects it, which the editor displays by drawing a red border around it.

When clicking on the Minus button:

  • if an item of this list is selected, this item will be removed from the list.
  • if no item is selected, the last item of the list will be removed.
  • if the list is empty, nothing happens.

When clicking on the Plus button:

  • if an item of this list is selected, a new item is inserted before the selected item.
  • if no item is selected, a new item is added at the end of the list.

External templates

One can add new templates to those already defined internally by Rezilla. New templates must be located in one of the Application Support folders. When Rezilla is launched, it creates, if they do not already exist, subfolders called Rezilla in the Application Support folder of the currently logged-in user and in the local system wide Application Support folder inside the Library folder. These locations are usually referred to as the user domain and the local domain respectively. Inside this folder is a Templates subfolder which is where the additional templates should be located for Rezilla to take them into account. Concretely, the folder paths are like this: 
    ~/Library/Application Support/Rezilla/Templates
    Library/Application Support/Rezilla/Templates
A template file is a file with a resource map containing resources of type TMPL. The resource map can be stored in the datafork or in the resource fork of the file.

Rezilla reads the contents of the Application Support/Rezilla/Templates folders at startup. If you add new template files there, you will have to quit and relaunch Rezilla in order to make them available to the application.

Local templates

Some resource files already contain TMPL resources for some types. In case you want to edit, in such a file, a resource having one of these types, Rezilla will automatically use the provided TMPL resource. Such a template is called a local template because it is found in the resource map you are currently editing and won't be available anymore as soon as this table is closed.

This can be useful if you transmit a non standeard resource to somebody and want to be sure he will be able to view it with the template editor. You just have to copy the TMPL resource in the resource file you transmit.

The template editor always looks for a TMPL resource in the current map before it tries to find an external or an internal template.
