Gabarits externes

On peut ajouter de nouveaux gabarits en complément de ceux qui sont définis de manière interne dans Rezilla. Les nouveaux gabarits doivent être placés dans un des dossiers Application Support. Lorsque l'application Rezilla est lancée, elle crée, si jamais ils n'existent pas déjà, des sous-dossiers appelés Rezilla dans le dossier Application Support de l'utilisateur courant et dans le dossier Application Support du système lui-même, à l'intérieur du dossier Bibliothèque. Dans ces dossiers sont créés des sous-dossiers appelés Templates (terme anglais pour gabarit) qui sont destinés à recevoir les fichiers de ressources comportant les nouveaux gabarits. Ces deux emplacements sont habituellement désignés comme le domaine de l'utilisateur et le domaine local respectivement. Concrètement, les chemins d'accès de ces deux dossiers sont: 
	~/Bibliothèque/Application Support/Rezilla/Templates
	Bibliothèque/Application Support/Rezilla/Templates

Un fichier de définition de gabarit est simplement un fichier de ressources contenant des ressources de type TMPL. Les ressources peuvent se trouver dans la branche de données ou dans la branche de ressources du fichier, indifféremment.

Rezilla lit le contenu des dossiers Application Support/Rezilla/Templates au moment de son démarrage. Si vous ajoutez de nouveaux gabarits par la suite, il faudra quitter l'application puis la relancer pour qu'ils soient pris en compte.

Depuis la version 1.1 de Rezilla, il est possible de placer dans le dossier Templates des alias pointant sur d'autres emplacements.
