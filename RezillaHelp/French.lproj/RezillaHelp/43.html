Export et import des tables de ressources

La commande Export Map du menu File permet de générer des fichiers en divers formats contenant les données relatives à une table de ressources. Les formats disponibles sont:
  • Texte
  • HTML
  • XML
  • DeRez

En formats XML et texte, il est possible de décider d'inclure les données binaires de chaque ressource ou pas: voir le panneau Export des Préférences. En format DeRez, les données binaires sont toujours incluses. En format Html, elles ne le sont pas.

En format texte, les données peuvent être représentées soit en encodage Base64 (qui est le format utilisé sous Mac OSX pour de nombreux types de fichiers XML tels que les fichiers de préférences ou les fichiers de listes de propriétés), soit sous forme hexadécimale (comme dans un éditeur hexadécimal). Ce choix peut être spécifié dans le panneau Export des Préférences. En format XML, les données sont toujours exportées dans l'encodage Base64. En format DeRez, elles le sont toujours en format hexadécimal.

DeRez est un format qui ne concerne que les développeurs: c'est le format utilisé par les outils Rez et DeRez. Les données de ressources sont exportées de façon identique à ce que produit l'outil DeRez lorsqu'aucun modèle particulier de ressources n'est spécifé (format data).

Pour des exports en format XML, la représentation des données est basée sur une DTD (Définition de Type de Document) définie dans le fichier Rezilla.dtd qui fait partie de la distribution de Rezilla. Chaque propriété est décrite au moyen d'une balise spécifique.

Voici un exemple de ce que produit, en XML, la commande Export: 

	<Type>
		<TypeCode>ALRT</TypeCode>
		<ResourcesArray>
			<Resource>
				<ResourceID>204</ResourceID>
				<ResourceName>Low Memory Warning</ResourceName>
				<ResourceFlags>20</ResourceFlags>
				<ResourceSize>14</ResourceSize>
				<ResourceData>
				AGgAggDNAegAzEREMAo=
				</ResourceData>
			</Resource>
		</ResourcesArray>
	</Type>
	<Type>
		<TypeCode>DITL</TypeCode>
		<ResourcesArray>
			<Resource>
				<ResourceID>204</ResourceID>
				<ResourceName></ResourceName>
				<ResourceFlags>20</ResourceFlags>
				<ResourceSize>120</ResourceSize>
				<ResourceData>
				AAEAAAAAAEcBIABbAVwEAk9LAAAAAAAKAEsAOgFciFdNZW1v
				cnkgaXMgZ2V0dGluZyBmdWxsLiBQbGVhc2UgdHJ5IHRvIGFs
				bGV2aWF0ZSB0aGUgcHJvYmxlbSBieSBjbG9zaW5nIHNvbWUg
				ZG9jdW1lbnRzLiDl
				</ResourceData>
			</Resource>
		</ResourcesArray>
	</Type>

La commande Import Map du menu File permet d'importer une table de ressources préalablement exportée en format XML. Le fichier d'import doit être un fichier XML valide et conforme à la DTD de Rezilla.
