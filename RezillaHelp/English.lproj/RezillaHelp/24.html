The Aete editor

This is an editor dedicated to AppleScript terminology resources (resources with type 'aete' or 'aeut'). These resources contain the dictionary of terms supported by an application for AppleScript scripting.

An 'aete' (or 'aeut') resource can contain several suites such as the Required Suite (with the Open, Quit etc. events), the Core Suite (with the Count, Make etc. events) and more application-specific suites. Each suite can contain information about four categories of objects: events, classes, comparison operators and enumerations. This is reflected by the Aete editor which has a unique window and lets you select a suite from a popup menu. A bar with four thumbnails called Events, Classes, Comparison Operators and Enumerations allows to select a category and displays a subpane with all the necessary information. If several objects are defined for a particular suite, the editor displays a slider to select one. If there is only one such object, no slider is dispalyed. If there are no such objects the corresponding subpane is not displayed.

When an Aete editor window is active a new menu named Terminology is inserted in the menu bar. It contains commands to add or remove a suite or to add or remove objects from the current suite. Depending on the category of the currently edited object some of the menu items can be disabled.

The Export Terminology command, in the File menu, lets you export the data of a terminology resource to a text file in format XML, DeRez, Sdef, HTML or text:

  • the HTML and text formats can be useful simply to make an AppleScript terminology resource human readable. It is sometimes necessary in an AppleScript script to know the raw codes for certain terms.

  • the XML format describes the entire data of the ressource according to a specific Xml format: a DTD defining the structure of the XML file is available in the project. Rezilla can also import this kind of files (see below) thus allowing for the data to be exchanged.

    Caveat: note that Rezilla does not do any conversion of special chars (like less than or greater than symbols, apostrophs, ampersands) to Xml entities. You'll have to edit the XML files if some strings contain such characters.

  • the DeRez format produces a file with a .r extension identical to what one obtains using the DeRez tool: this file is ready to be recompiled in any project (with the Rez tool for instance).

  • the Sdef format (Scripting Definition File) describes the data of a terminology resource according to the new format defined by Apple. See its definition in the file /System/Library/DTDs/sdef.dtd and the manual page (type "man sdef" in a Terminal window). Note that the Sdef format does not retain all the information contained in an Aete resource: the translation from Aete to Sdef is necessarily lossy. Rezilla does its best to make this conversion accurate.

Important: The XML, Sdef and HTML export files are UTF-8 encoded. The text and DeRez export files are MacRoman encoded.

The Import Terminology command, in the File menu, lets you import data from an XML file into a terminology resource. To import an XML file, you can, for instance, create a new 'aete' resource, edit it and select the Import Terminology command in the Terminology menu. You can also import an XML file into an already existing resource : the data imported will be appended to the resource. The XML file must be in the format defined in the DTD included in the project (aete.dtd, not to be confused with sdef.dtd).
