Template Editor

Rezilla is able to parse resources using descriptive templates and to build a graphical interface out of it, allowing to easily edit all the elements of the resource. For instance, the following image shows how Rezilla would edit a MPSR resource containing font and size information about a document.

There are three kinds of locations where TMPL resources can be found. They are described in details in the following subsections:

  • internal templates are predefined templates stored inside the application. Rezilla ships with more than 360 predefined templates covering the most currently used resource types;
  • external templates are additional templates that one can add to one of the Application Support folders;
  • local templates are TMPL resources present in the currently edited resource map.

When Rezilla attempts to edit a resource using a template, it looks for the appropriate template in all these locations in the following order: local, external, internal. You can thus override the definition of an internal template by putting a new definition in the external locations.

External templates

One can add new templates to those already defined internally by Rezilla. New templates must be located in one of the Application Support folders. When Rezilla is launched, it creates, if they do not already exist, subfolders called Rezilla in the Application Support folder of the currently logged-in user and in the local system wide Application Support folder inside the Library folder. These locations are usually referred to as the user domain and the local domain respectively. Inside this folder is a Templates subfolder which is where the additional templates should be located for Rezilla to take them into account. Concretely, the folder paths are like this: 
    ~/Library/Application Support/Rezilla/Templates
    Library/Application Support/Rezilla/Templates
A template file is a file with a resource map containing resources of type TMPL. The resource map can be stored in the datafork or in the resource fork of the file.

Rezilla reads the contents of the Application Support/Rezilla/Templates folders at startup. If you add new template files there, you will have to quit and relaunch Rezilla in order to make them available to the application.

Local templates

Some resource files already contain TMPL resources for some types. In case you want to edit, in such a file, a resource having one of these types, Rezilla will automatically use the provided TMPL resource. Such a template is called a local template because it is found in the resource map you are currently editing.

This can be useful if you transmit a non standeard resource to somebody and want to be sure he will be able to view it with the template editor. You just have to copy the TMPL resource in the resource file you transmit.

The template editor always looks for a TMPL resource in the current map before it tries to find an external or an internal template.
