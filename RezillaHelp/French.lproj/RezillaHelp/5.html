Types de remplacement

Certains types de ressource ont des structures identiques. Par exemple, une ressource de type kcs4 décrivant une petite icone de clavier utilise le même format qu'une ressource de type ics4. Ces types sont donc interchangeables du point de vue des éditeurs, l'un pouvant être édité au moyen d'un éditeur de l'autre. Rezilla connaît un certain nombre de situations de ce type et opère la substitution automatiqument lorsque c'est nécessaire. Par exemple, s'il ne trouve pas de gabarit pour une ressource itl0, il utilisera celui des ressources INTL à la place.

On peut aussi forcer Rezilla à se servir d'un type à la place d'un autre. Il faut pour cela utiliser la commande Save as type... qui se trouve dans le menu Resource. Elle affiche un petit dialogue dans lequel on peut spécifier le type de remplacement soit à partir d'un menu, soit en le tapant directement dans le champ de saisie.
