Plugins (éditeurs externes)

Depuis la version 1.1, Rezilla a mis en place un support pour l'addition d'éditeurs externes appelés plugins. Un plugin est une sorte d'extension qui apporte de nouvelles capacités d'édition à Rezilla. Par exemple, il pourrait y avoir un plugin définissant une belle interface graphique pour les ressources 'WIND' permettant de les visualiser et de les manipuler aisément: Rezilla possède déjà un gabarit pour éditer les ressources 'WIND' mais un plugin permettrait d'introduire une interface graphique. Rezilla fournit un kit de développement (SDK, Software Development Kit) pour permettre à des développeurs ou des éditeurs de logiciels d'écrire ce type de plugins.

Un plugin peut supporter plusieurs types de ressources. La version 1.1 de Rezilla fournit par exemple un plugin permettant d'afficher et de manipuler des ressources d'image. Il supporte les formats d'images suivants:

JPEG, TIFF, GIF, PNG, BMP.

Un plugin peut être ajouté ou supprimé. On peut contrôler la liste des plugins très facilement depuis le Finder comme le montre l'image suivante:

  • sélectionner Rezilla dans le Finder

  • sélectionner la commande Lire les informations dans le menu Fichier (ou presser commande-I)

  • dans la fenêtre d'informations, il y a une section appelée Modules externes que l'on peut ouvrir afin d'afficher la liste des plugins actuellement inclus dans Rezilla

  • les boutons Ajouter et Supprimer situés sous la liste permettent respectivement d'installer un nouveau plugin ou de désinstaller un plugin existant. Chaque plugin possède une case à cocher: on peut la dé-cocher si l'on souhaite désactiver un plugin temporairement plutôt que de le supprimer.

Au cours du lancement de Rezilla, l'application examine les plugins présents afin de détecter quel(s) type(s) chacun d'eux est susceptible d'éditer. Cela signifie que si l'on ajoute ou supprime un plugin,il faut quitter Rezilla et le relancer pour que les modifications soient prises en compte.

L'édition d'une ressource au moyen d'un éditeur défini dans un plugin se fait automatiquement lorsque l'utilisateur invoque la commande Edit: l'application cherche tout d'abord un éditeur interne correspondant au type de la ressource à éditer; si aucun éditeur interne n'est trouvé, elle recherche si un plugin convient; si aucun plugin n'est disponible, Rezilla tente alors d'appeler l'éditeur de gabarits et, en cas d'échec, l'éditeur hexadécimal est invoqué en dernier recours.

On peut aussi invoquer directement l'édition par plugin avec la commande Edit with Plugin qui se trouve dans le menu Resources, comme on peut le voir sur l'image ci-dessous: la commande fait l'impasse sur la recherche d'un éditeur interne. Un raccourci-clavier équivalent est option-contrôle-commande-E (presser E tout en maintenant les touches option, contrôle et commande enfoncées).

Les dévelopeurs souhaitant créer un plugin pour Rezilla trouveront les informations nécessaires dans la section Écrire un Plugin pour Rezilla.
