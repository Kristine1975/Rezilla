Introduction

Les fichiers Macintosh ont traditionellement deux branches appelées respectivement la branche de données et la branche de ressources. À l'origine les ressources étaient toujours stockées dans la branche de ressources. Depuis l'avènement du système OSX, il est désormais possible de stocker les ressources dans la branche de données d'un fichier. Rezilla permet d'éditer, créer, modifier, dupliquer, détruire des ressources contenues dans n'importe quelle branche, de les copier, couper et coller d'une table de ressources dans une autre, de faire glisser des ressources etc. Une table de ressources peut aussi être exportée en format texte, HTML, XML ou DeRez. Rezilla peut enfin comparer deux tables de ressources.

Le projet Rezilla entend être un projet à la fois extensible et collaboratif:

  • extensible signifie qu'il supportera une architecture d'extensions (plugins) permettant de définir et d'ajouter des éditeurs externes spécifiques à un type particulier de ressource
  • collaboratif signifie que tous les programmeurs souhaitant collaborer au projet et proposer des contributions de code sont les bienvenus.
