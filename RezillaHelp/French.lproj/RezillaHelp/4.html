Description des articles des menus

Menu Fichier

    New Map
    Permet de créer un nouveau fichier de ressources. Un dialogue standard permet d'indiquer le nom et l'emplacement du nouveau fichier de ressources à créer; deux boutons-radios permettent de specifier si la table de ressources doit être stockée dans la branche de données ou dans la branche de ressources du fichier.

    Open...
    Présente un dialogue standard pour ouvrir une table de ressources. Dans ce dialogue, trois boutons-radios permettent de specifier la branche dans laquelle rechercher les ressources. Les options proposées sont:
    • "any fork": dans ce cas, Rezilla cherche en premier une table de ressources dans la branche de données. S'il n'en trouve aucune, il cherche alors dans la branche de ressources.
    • "data fork": recherche une table de ressources seulement dans la branche de données du fichier
    • "resource fork": recherche une table de ressources seulement dans la branche de ressources du fichier
    On peut aussi sélectionner la case à cocher Read Only: si elle est cochée, la table sera ouverte en mode lecture seule et toutes les commandes d'édition seront désactivées.

    Recent items
    Ce sous-menu contient les noms des dix plus récentes tables de ressources ouvertes dans Rezilla. Si on sélectionne un article de ce menu, la table de ressources correspondante est ouverte.

    Close
    Ferme la fenêtre se trouvant au premier-plan.

    Revert
    Cette commande permet de restaurer la dernière version sauvegardée sur disque dans le cas où la table courante a été modifiée et que l'on ne veut pas conserver ces modifications.

    Save
    Sauvegarde les modifications sur disque.

    Save As...
    Présente un dialogue standard pour spécifier un fichier dans lequel stocker la table de ressources courante. Trois boutons-radios permettent de spécifier dans quelle branche stocker la table. Les options proposées sont:
    • "same fork": utiliser la même branche que la table courante.
    • "data fork": sauvegarder dans la branche de données.
    • "resource fork": sauvegarder dans la branche de ressources.

    Export...
    Cette commande permet d'exporter la structure et le contenu d'unz table de ressources dans des fichiers selon divers formats. Les formats disponibles sont:
    • Text
    • HTML
    • XML
    • DeRez
    En format XML, les données binaires de chaque ressource sont encodées en encodage Base64 comme il est usuel en XML lorsque l'on a besoin de stocker des données binaires (les fichiers plist sous OSX par exemple utilisent cet encodage). En format text au contraire, les données binaires de chaque ressource sont encodées sous forme hexadécimale (chaque octet est représenté par deux nombres hexadécimaux comme dans un éditeur hexadécimal).

    DeRez est un format qui ne concerne que les développeurs: c'est le format utilisé par les outils Rez et DeRez. Les données de ressources sont exportées de façon identique à ce que produit l'outil DeRez lorsqu'aucun modèle particulier de ressources n'est spécifé (format data).

Menu Edit

    Copy / Cut / Paste
    Les fonctions de Copier, Couper et Coller peuvent avoir différentes significations selon le contexte. Voir la section Copier, Couper et Coller plus loin.

    Clear
    Détruit la sélection. Cette fonction peut avoir différentes significations selon le contexte. Si une table de ressources est active et que des ressources y sont sélectionnées, elles sont supprimées de la table; si une fenêtre d'éditeur hexadécimal être est active et que du texte est sélectionné dans un des cadres, cette sélection est détruite.

    Select all
    Cette commande permet de tout sélectionner: toutes les ressources dans une table de ressource, tout le texte dans une fenêtre de texte etc.

    Find...
    Dans l'éditeur hexadécimal, cette commande présente un dialogue dans lequel on peut entrer une chaîne de caractères à rechercher. Deux boutons-radios permettent de spécifier dans quel panneau la recherche doit s'opérer. Si l'on recherche dans le panneau de droite, une case à cocher permet en plus de spécifier si la recherche doit tenir compte de la casse des caractères.

    Find again
    Cette commande recherche, depuis position courante, la chaîne entrée dans le dialogue de recherche (appelé par la commande Find). Elle utilise les mêmes critères que ceux qui ont été sélectionnés dans ce dialogue. Si le dialogue Find contient une chaîne vide, la commande est désactivée.

    Compare maps...
    Cette commande permet de comparer deux tables de ressources. Cela peut être utile si l'on veut voir les changements entre deux versions successives d'une application ou d'un fichier. Les résultats sont affichés dans une fenêtre comportant trois listes: la liste de gauche contient les ressources trouvées dans la première table de ressources et absentes de la seconde, la liste de droite contient les ressources trouvées dans la seconde table de ressources et absentes de la première. La liste du milieu contient les ressources communes aux deux tables de ressources et qui diffèrent selon l'un des critères de comparaison. En cliquant sur une ressource dans cette liste, les données hexadécimales de la ressource sont affichées dans deux panneaux parallèles au bas de la fenêtre afin de les comparer: à gauche, les données provenant de la première table de ressources, et à droite, les données provenant de la deuxième. Les données peuvent être vues ou bien en format hexadécimal ou bien en format texte au moyen des deux boutons-radios au bas de la fenêtre.

Menu Resource

    New resource
    Permet de créer une nouvelle ressource dans la table de ressources courante. Une fenêtre de dialogue permet de spécifier le type, le numéro ID, le nom et les attributs de la nouvelle ressource. Une ressource est alors créée ne contenant aucune donnée: la ressource est créée en mémoire et il faudra sauvegarder le fichier pour la rendre définitive.

    Edit
    Appelle l'éditeur hexadécimal pour toutes les ressources sélectionnées dans la table de ressources courante. C'est équivalent à double-cliquer sur une ressource dans une table de ressources.

    Get Info
    Appelle la fenêtre Inspecteur pour le première ressource sélectionnée dans la table de ressources courante. C'est équivalent à cliquer sur une ressource dans une table de ressources en maintenant la touche option enfoncée.

    Duplicate
    Duplique toutes les ressources sélectionnées dans la table de ressources. Cette commande attribue un numéro unique d'identification (ID) aux ressources dupliquées. On peut changemodifer ces valeurs par la suite dans la fenêtre Inspecteur.

    Remove
    Supprime toutes les ressources sélectionnées de la table de ressources.

Menu Fenêtres

Ce menu comporte la liste les fenêtres couramment ouvertes dans Rezilla (tables de ressources, fenêtres d'édition etc.). En sélectionant un article dans ce menu, on amène au premier-plan la fenêtre correspondante.

Menu Aide

Le menu Aide contient une commande appelée "Aide Rezilla"qui permet d'accéder directement à la documentation de Rezilla et de la lire en utilisant le Visualisateur d'Aide du Système MacOS.
