Exporting

The exporting functionality lets you generate a file containing all the data of the resource map (all resources of all types, with their ID, name, attributes etc.) in various formats. Available formats are:
  • Text
  • HTML
  • XML
  • DeRez

In Text and XML formats, it is possible to decide to include the binary data for each resource or not: see the Preferences section. The binary code for each resource is encoded either in Base64 encoding (which is the format used on MacOSX for many other types of binary data stored in XML files, like for instance preferences files, property lists etc.), or in hexadecimal representation. This is a choice which can be specified in the Preferences too.

For XML export, there are two ways of representing the data, based on two different DTDs (Document Type Definitions). There is an elements-based DTD in which all the properties are represented by a specific key and an attributes-based DTD in which most of the properties are represented as attributes for the basic type and resource elements.
